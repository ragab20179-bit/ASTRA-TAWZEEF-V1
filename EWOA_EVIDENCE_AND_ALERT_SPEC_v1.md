# EWOA Evidence and Alert Specification v1.0

This document defines the schema for the Emergency Workaround Authorization (EWOA) evidence bundle, the alert definitions, and the severity rules for Non-Conformance Reports (NCRs).

## 1. Evidence Bundle Schema

The EWOA evidence bundle is a JSON object that contains all the information related to an EWOA fire drill. The schema for the evidence bundle is as follows:

```json
{
  "schema": "EWOA_EVIDENCE_BUNDLE",
  "version": "1.0",
  "generated_at": "<ISO 8601 timestamp>",
  "drills": [
    {
      "name": "<drill name>",
      "status": "<PASS|FAIL>",
      "metrics": {
        "<metric_name>": "<metric_value>"
      }
    }
  ],
  "metrics": {
    "p95_latency_ms": "<value>",
    "error_rate_percent": "<value>",
    "denial_rate_percent": "<value>"
  },
  "alerts": [
    {
      "name": "<alert_name>",
      "severity": "<LOW|MEDIUM|HIGH|CRITICAL>",
      "details": "<alert_details>"
    }
  ],
  "result": "<PASS|FAIL>"
}
```

## 2. Alert Definitions

Alerts are generated by the EWOA Signal Evaluation Layer based on the metrics collected during the fire drill. The following alerts are defined:

| Alert Name | Metric | Threshold | Severity |
| --- | --- | --- | --- |
| High Latency | p95_latency_ms | > 100ms | HIGH |
| High Error Rate | error_rate_percent | > 10% | CRITICAL |
| High Denial Rate | denial_rate_percent | > 5% | MEDIUM |

## 3. NCR/CAPA Severity Rules

Non-Conformance Reports (NCRs) and Corrective and Preventive Action (CAPA) reports are generated based on the severity of the alerts triggered during the EWOA fire drill. The following severity rules are defined:

| Severity | NCR/CAPA Trigger |
| --- | --- |
| CRITICAL | Immediate |
| HIGH | If the alert persists for more than 1 minute |
| MEDIUM | If the alert persists for more than 5 minutes |
| LOW | No automatic NCR/CAPA generation |

## 4. EWOA Signal Evaluation Layer

The EWOA Signal Evaluation Layer is a conceptual component that is responsible for consuming metrics from the k6 load tests and generating alerts based on the defined alert definitions. This layer is independent of the test tool and provides a clear separation between signal generation and signal evaluation.

-   **Inputs:**
    -   p95 latency (ms)
    -   Error rate (%)
    -   Denial rate (%)
-   **Outputs:**
    -   Alarm events (High Latency, High Error Rate, High Denial Rate)
